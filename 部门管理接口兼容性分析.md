### 3.2 关键字段缺失问题

**问题描述**：
- 两个系统的DeptDTO都缺少 `organizationId` 字段
- 但实体类（Dept.java）和VO（DeptVO）都包含此字段
- 这会导致创建部门时无法指定组织，造成数据完整性问题

**影响范围**：
- 创建部门接口无法指定组织ID
- 更新部门接口无法修改组织ID
- 可能导致数据混乱和组织隔离失效

**建议修复方案**：

```java
// DeptDTO 需要补充 organizationId 字段
@Schema(description = "组织ID", requiredMode = Schema.RequiredMode.REQUIRED)
@NotNull(message = "组织ID不能为空")
private Long organizationId;
```

| 字段 | structure-org 校验 | structure-admin 校验 | 差异影响 |
|------|-------------------|---------------------|----------|
| name | ✅ `@NotBlank(message = "部门名称不能为空")` | ❌ 无校验 | org更严格 |
| parentId | ✅ `@NotNull(message = "父部门ID不能为空")` | ✅ `@NotNull(message = "父部门ID不能为空")` | 一致 |

## 九、平滑升级建议方案

### 9.1 方案一：路径隔离（推荐）

**方案描述**：
```
structure-org:  /api/org/dept
structure-admin: /api/admin/dept
```

**优点**：
- ✅ 完全隔离，无路径冲突
- ✅ 保留各自业务逻辑
- ✅ 最小改动成本
- ✅ 支持独立部署和升级

**缺点**：
- ⚠️ 需要前端适配新路径
- ⚠️ 需要更新API文档

**实施步骤**：
1. 修改structure-org的DeptController路径为 `/api/org/dept`
2. 修改structure-admin的DeptEndpoint路径为 `/api/admin/dept`
3. 更新前端API调用路径
4. 更新Swagger文档路径

**代码修改示例**：

```java
// structure-org
@RestController
@RequestMapping("/api/org/dept")
@Tag(name = "组织架构-部门管理", description = "组织架构部门管理接口")

    // ...
}

// structure-admin
@RestController
@RequestMapping(value = "/api/admin/dept")

    // ...
}
```

**优点**：

**缺点**：
- ⚠️ 需要实现版本控制机制
- ⚠️ 改动成本较高
- ⚠️ 需要维护多版本接口

### 9.3 方案三：功能合并（复杂）

**方案描述**：
将两个系统合并为一个统一的部门管理服务

**优点**：
